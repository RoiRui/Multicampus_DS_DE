## 📂 클래스와 객체(Class and Objects)

#### 📌 객체 지향 프로그래밍이란?
- 객체 지향 프로그래밍에서는 모든 데이터를 객체로 취급
- **객체의 상태(State)와 행동(Behavior)을 구체화하는 형태의 프로그래밍을 객체 지향 프로그래밍이라고 한다**

#### 📌 클래스란?
- **실세계의 것(사물)을 모델링하여 속성(Attribute)와 동작(Method)를 갖는 데이터 타입**
- 객체를 만들어 내기 위한 설계도와 같은 개념을 클래스(Class)라고 한다
- **다루고자 하는 데이터(변수)와 데이터를 다루는 연산(함수)를 하나로 캡슐화(Encapsulation)하여 클래스로 표현**
- 모델링에서 중요시 하는 속성에 따라 클래스의 속성과 행동이 각각 달라진다
- Python에서의 string, int, list, dict, ... 모두가 클래스로 존재하고 있다

#### 📌 클래스 선언
- 객체를 생성하기 위해서는 객체의 모체가 되는 클래스를 미리 선언해야 한다

```python
class 클래스명:
  클래스 생성자
  클래스 속성
  클래스 메소드
```

#### 📌 객체란?
- 클래스로 생성되어 구체화된 객체(인스턴스)
- 실제로 클래스가 인스턴스화 되어 메모리에 상주하는 상태를 의미

#### 📌 인스턴스란?
- 메모리에 할당된 객체를 의미
- 생성된 인스턴스는 독립된 메모리 공간에 저장된 자신만의 필드를 가질 수 있다

```python
객체명 = 클래스명()
객체명 = 클래스명(인수1, 인수2, ...)
```

#### 📌 __init__(self)
- 생성자, 클래스 인스턴스가 생성될 때 호출된다
- self 인자는 항상 첫번재에 오며 자기 자신을 가리킨다
- 이름이 꼭 self일 필요는 없지만 관례적으로 사용한다
- 생성자에서는 해당 클래스가 다루는 데이터를 정의 : 이 데이터를 멤버 변수(Member Variable) 또는 속성(Attribute)라고 한다

#### 📌 self
- 파이썬의 메소드는 항상 첫번째 인자로 self를 전달
- self는 현재 해당 메소드가 호출되는 객체 자신을 가리킨다
- C++, C#, Java에서의 this에 해당한다

#### 📌 메소드란?
- 멤버 함수라고도 하며 해당 클래스의 Object에서만 호출이 가능하다
- 메소드는 객체 레벨에서 호출되며 해당 객체의 속성에 대한 연산을 행한다
- obj.method() 형태로 호출된다

#### 📌 Method Type
- Instance Method : 객체로 호출(일반 메소드)
  - 메소드는 객체 레벨로 호출되기 때문에 해당 메소드를 호출한 객체에만 영향을 미친다
- Class Method : @staticmethod 사용
  - 클래스 메소드의 경우 클래스 레벨로 호출되기 때문에 클래스 멤버 변수만 변경 가능

```python
※ Class Method

class Math:
  @staticmethod
  def add(a, b):
    return a + b
  @staticmethod
  def multiply(a, b):
    return a * b

Math.add(10, 20)
Math.multiply(10, 20)
```

#### 📌 상속(Class Inheritance)
- 기존에 정의해둔 클래스의 기능을 그대로 물려받을 수 있다
- 기존 클래스에 기능 일부를 추가하거나 변경하여 새로운 클래스를 정의
- 코드를 재사용할 수 있게 된다
- 상속 받고자 하는 대상인 기존 클래스는 Parent, Super, Base 클래스라고 부른다
- 상속 받는 새로운 클래스는 Child, Sub, Derived 클래스라고 부른다
- 의미적으로 is-a 관계를 가짐

#### 📌 Method Override
- 부모 클래스의 메소드를 재정의
- 하위 클래스(자식 클래스)의 인스턴스로 호출 시 재정의된 메소드가 호출된다

#### 📌 super
- 하위 클래스(자식 클래스)에서 부모 클래스의 메소드를 호출할 때 사용

```python
class Person:
  def __init(self, name, age)
    self.name = name
    self.age = age
  
  def eat(self, food):
    print('{}은 {}를 먹습니다.'.format(self.name, food))
  
  def sleep(self, minute):
    print('{}은 {}분동안 잡니다.'.format(self.name, minute))
  
  def work(self, minute)
    print('{}은 {}분동안 일합니다.'.format(self.name, minute))
    
class Student(Person): ➝ Class Inheritance
  def __init__(self, name, age):
    self.name = name
    self.age = age
  
  def work(self, minute): ➝ Method Override
    super().work(minute) ➝ Super
    print('{}은 {}분동안 공부합니다.'.format(self.name, minute))
  
class Employee(Person): ➝ Class Inheritance
  def __init__(self, name, age):
    self.name = name
    self.age = age
    
  def work(self, minute): ➝ Method Override
    super().work(minute) ➝ Super
    print('{}은 {}분동안 업무를 합니다.'.format(self.name, minute))

lee = Student('Lee', 19)
lee.eat('NOODLE')
lee.sleep(60)
lee.work(30) ➝ Override된 Method

bob = Employee('bob', 25)
bob.eat('BBQ')
bob.sleep(30)
bob.work(60) ➝ Override된 Method
```

#### 📌 클래스가 꼭 필요한가?
- C 언어에는 클래스가 없다 (클래스가 없어도 프로그램을 충분히 만들 수 있다)
- 파이썬 프로그램 중 클래스를 사용하지 않은 잘 만든 프로그램도 상당히 많다
- 프로그램 작성을 위해 꼭 필요한 요소는 아니다
- **단, 클래스를 잘 활용하면 프로그래머가 얻을 수 있는 이익이 많다**
